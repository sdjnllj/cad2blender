# CAD 到 Blender 自动化建模工作流

## 概述

用户使用 AutoCAD 绘制建筑平面图和立面图（DXF 格式），通过 Python 脚本自动在 Blender 中生成 3D 建筑外立面模型（含窗框）。

## 工作流

```
CAD 绘图 (.dxf)  →  parse_dxf.py  →  data.json  →  build_from_json.py  →  Blender 3D  →  export_fbx.py  →  .fbx
```

### 1. 用户操作
- 在 AutoCAD 2021 中按规范绘图
- 导出 DXF 文件（AutoCAD 2010+ 格式）
- 运行解析脚本生成 JSON

### 2. 解析
```bash
python parse_dxf.py <图纸.dxf> [输出.json]
```

### 3. 建模（通过 MCP Blender 工具）
在 Blender 中执行：
```python
import sys, importlib
sys.path.insert(0, r'项目路径')
import build_from_json
importlib.reload(build_from_json)
build_from_json.build("路径/data.json")
```

### 4. FBX 导出
在 Blender 中执行：
```python
import export_fbx
export_fbx.export("输出路径.fbx")                      # 默认导出 "Building" 父对象
export_fbx.export("输出.fbx", building_name="Building") # 或指定父对象名
```
导出参数针对 3ds Max 优化（Z-up、单位缩放、四边形网格）。

## CAD 图层规范

| 图层名 | 用途 | 图元类型 | 说明 |
|--------|------|---------|------|
| `Plan_Walls` | 墙体平面定位 | LINE（双线） | 每对平行线 = 一面墙，壁厚 = 双线间距 |
| `Plan_Walls` | 墙编号标签 | MTEXT | 在墙体旁标注编号（W1, W2, ...），自动关联到最近墙体 |
| `Plan_Columns` | 柱子平面 | LWPOLYLINE（闭合） | 矩形/异形柱轮廓 |
| `Elev_<编号>` | 立面图 | LINE 外轮廓 + LWPOLYLINE/LINE 洞口 | 如 `Elev_W6` 对应编号 W6 的墙体 |
| `Windows` | 窗框正视图 | LINE | 画在对应墙的立面图范围内（坐标重叠），AI 自动归属 |

---

## 立面图（Elev_\<编号\>）规范

### 外轮廓画法
- **4 条 LINE 首尾相连**（闭合环）= 立面外轮廓（推荐）
- AI 用连通性分析找最大面积闭合环作为外轮廓，其余小环自动识别为洞口
- 底边可省略（AI 自动退回贪心拼接，兼容不完整轮廓）

### 洞口（门窗洞）画法（任选其一）
| 方式 | 命令 | 图元 | 说明 |
|------|------|------|------|
| 矩形 | `RECTANGLE` / `REC` | LWPOLYLINE（闭合） | **推荐**，最快 |
| 多段线 | `PLINE` | LWPOLYLINE（闭合） | 任意形状洞口 |
| 4条直线 | `LINE` × 4 | 形成闭合环 | 也支持，自动识别 |

### 立面图位置规则
- 立面图可放在 CAD 中**任意位置**，不要求和平面图重叠
- 立面的 X 轴 = 沿墙方向，Y 轴 = 高度方向
- 立面图可覆盖多层（一楼到顶一次画完）

## Windows 图层规范（窗框）

### 核心原则：画在洞口里，不需要标签
- 把窗框正视图**直接画在对应墙立面图的洞口位置内**（CAD 绝对坐标落在该 Elev_ 的范围内）
- AI 通过**空间包含检测**（窗线中点落在哪个 Elev_ 的包围盒内）自动归属到对应墙和洞口
- **不需要任何标签**，零额外标注工作

### 窗框 CAD 绘图规范
每扇窗用两层矩形 + 中梃线表达：

```
外矩形（LINE × 4）── 与洞口对齐，定义窗框外边界
内矩形（LINE × 4）── 内缩约框料宽度，定义玻璃区域边界
中梃/横档（LINE）── 竖向或横向分格线（成对出现，间距=框料宽度）
```

- 框料宽度（内外矩形之差）由 AI 从坐标自动计算，无需标注
- 平行线对间距即框料宽度（典型值 60mm）
- 中梃只画需要的区域（如推拉窗竖梃只画下半段）

### 窗框 3D 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `frame_offset_mm` | 100 | 窗框前脸距外立面的距离（内缩量） |
| `frame_depth_mm` | 60 | 窗框进深 |

可在 JSON meta 中覆盖：
```json
"meta": {
  "window_defaults": {
    "frame_offset_mm": 100,
    "frame_depth_mm": 60
  }
}
```

## 建模逻辑

### 墙体定位（核心原则）
- **平面图外轮廓 LINE 端点 = 墙体精确位置**，不做中心线计算或转角延伸
- 每对平行线中，离建筑重心更远的那条即为外立面线
- 外立面线端点直接用于 3D 定位，长度即墙体实际长度
- 壁厚 = 双线间距（从 LINE 坐标直接读取）

### 有立面图的墙体
- 使用 grid-cut 算法生成纯四边形网格（无布尔运算）
- 立面外轮廓 → 墙体外面，洞口 → 窗洞切穿
- 外立面定位在外轮廓线上，墙体向内（重心方向）延伸

### 无立面图的墙体
- 自动继承所有立面的最大高度作为默认高度
- 生成为简单矩形盒体（无洞口），同样用外轮廓线定位

### 柱子
- 按平面轮廓挤出，高度继承默认高度

### 窗框建模算法（格栅法）
1. **格栅切割**：以窗线所有端点 X/Y 坐标建格
2. **固识别**：格子最短边 ≤ 200mm 且两侧均有窗线部分覆盖 → 框料格
3. **合并**：同轴且间距 ≤ 200mm 的相邻格 → 合并为一根连续框料条
4. **延伸**：接触外边界的竖框延伸到全高；横框延伸到全宽（填满四角）
5. **裁剪**：竖框（高度>宽度）裁剪到横框之间（消除四角重叠面）

生成命名规则：`Win_<墙号>_o<洞口序号>_b<框料条序号>`

### 坐标映射
- 单位：毫米（mm）→ Blender 中自动转换为米（÷1000）
- 墙体外面方向：自动判断（远离建筑重心的一侧）

## 设计决策

- **纯四边形网格**：不使用布尔运算，避免伪影
- **各墙独立建模**：转角处自然重叠，不处理斜接（用于外立面效果图）
- **不需要楼层概念**：用户直接在立面图中画好总高度和各层洞口
- **两步流水线**：解析和建模分离，方便单独调试 JSON 中间数据
- **Windows 层空间归属**：无标签，用坐标包含关系自动归属，对人工零额外负担
- **横框优先原则**：四角处横框（底/顶）占满宽度，竖框（左/右）让位，无重叠面
- **材质按类型预建**：`Mat_Wall`（浅灰）赋给全部墙体和柱子，`Mat_Window`（浅蓝灰）赋给窗框；导出 FBX 后在 3ds Max 中只需替换这两个材质槽
